React

React uses one file to build a component. Every component requires a `render` method that contains the layout of the component. `render` accepts one root node with children. More 

            
              <!-- src/scripts/chat-container.js -->
              export default React.createClass({
                render() {
                  return (
                    <div className="chat-room">
                      <h1>React Chat</h1>
                      ...
                    </div>
                  );
                }
              });
            
          
            
              <!-- output -->
              <div class="chat-room">
                <h1>React Chat</h1>
              </div>

Ember

Ember components are built with two files: a view object and a template. Ember component is the root node and the template contains its contents. More 

            
              <!-- app/components/chat-container.js -->
              export default Ember.Component.extend({
                tagName: 'div', // default `div`
                classNames: ['chat-room']
              });

              <!-- app/templates/components/chat-container.hbs -->
              <h1>Ember Chat</h1>
            
          
            
              <!-- output -->
              <div id="ember382" class="ember-view chat-room">
                <h1>Ember Chat</h1>
              </div>

If you name a component `components/hello-world`, you will be able to use in other templates with `{{hello-world}}`

React

React displays data through `props` and `state`. State should contain data that a component's event handlers may change to trigger a UI update More 
            
              // <ChatContainer title="React Chat"/>

              export default React.createClass({
                getInitialState() {
                  return { show: false };
                },
                getDefaultProps() {
                  return { title: 'Default Title' };
                },
                render() {
                  let classname = 'chat-room';
                  classname += this.state.show ? ' show' : ' hide';
                  return (
                    <div className={classname}>
                      <h1>{this.props.title}</h1>
                    </div>
                  );
                }
              });
            
          
            
              <!-- output -->
              <div class="chat-room hide">
                <h1>React Chat</h1>
              </div>

Ember

Ember uses properties that can be used in the template for display. Properties can be set in the component or passed in. More 
            
              // {{chat-container title="Ember Chat"}}
              export default Ember.Component.extend({
                classNames: ['chat-room'],
                classNameBindings: ['isVisible:show:hide'],
                title: 'Default Title',
                isVisible: false
              });

              <!-- app/templates/components/chat-container.hbs -->
              <h1>{{title}}</h1>
            
          
            
              <!-- output -->
              <div id="ember382" class="ember-view chat-room hide">
                <h1>Ember Chat</h1>
              </div>

React has various methods that are executed at specific points in a component's lifecycle. More 
            
      export default React.createClass({
        // Invoked immediately before the initial rendering occurs
        componentWillMount() {},

        // Invoked immediately after the initial rendering occurs
        componentDidMount() {},

        // Invoked when a component is receiving new props
        componentWillReceiveProps() {},

        // Invoked before rendering when new props or state are being received
        shouldComponentUpdate() {},

        // Invoked immediately before rendering when new props or state are being received
        componentWillUpdate() {},

        // Invoked immediately after the component's updates are flushed to the DOM
        componentDidUpdate() {},

        // Invoked immediately before a component is unmounted from the DOM
        componentWillUnmount(),
        render() { ... }
      });

Ember

Ember has various methods that are executed at specific points in a component's lifecycle. More 
            
          export default Ember.Component.extend({
            classNames: ['chat-room'],
            init() {},
            didReceiveAttrs() {}, // runs after init
            willRender(), // runs before render
            didInsertElement() {}, // after renders its backing HTML element into the DOM
            didRender() {}, // after render

            didUpdateAttrs() {}, // when the attributes of a component have changed
            willUpdate() {}, // runs before update
            didUpdate(), // runs after update

            willDestroyElement() {}, // before removing itself from the DOM
            willClearRender() {},
            didDestroyElement() {} // after removal from DOM
          });
